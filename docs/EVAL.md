## LIBERO Evaluation Script Flow

### Overview

The LIBERO evaluation script (`run_libero_eval.py`) evaluates a trained Cosmos Policy model on LIBERO benchmark tasks. It runs episodes in simulation, queries the policy for actions, and tracks success rates.

## get_action() Function Flow

### Overview

The `get_action()` function is the core of the policy inference system. It converts observations into action predictions using a diffusion-based policy model. The function processes multimodal inputs (images, proprioception, text) and generates an action chunk through latent diffusion.

## Key Concepts

### 1. Best-of-N Search
- The evaluation script queries the model N times with different seeds
- Compares predicted values from each query
- Selects the action chunk with the highest predicted value
- This improves success rate by exploring multiple action possibilities

### 2. Action Chunking
- Model predicts 16 actions at once (chunk_size=16)
- Executes 16 steps open-loop (num_open_loop_steps=16)
- Then requeries for the next chunk
- This balances between efficiency and adaptability

### 3. Parallel vs Serial Inference
- **Serial**: Queries sequentially (default, simpler)
- **Parallel**: Uses multiple GPUs to query simultaneously (faster for best-of-N)
- Parallel mode requires multiple GPUs and more memory

### 4. Latent Sequence Structure
- Fixed format with specific indices for:
  - Current state (proprio, images)
  - Action (to be generated)
  - Future state (predicted)
  - Value (task completion probability)
- Each element duplicated 4x for temporal compression

### 5. Diffusion Generation
- Uses iterative denoising process
- Starts with noise, gradually refines to clean action
- Conditions on text instruction and current state
- Generates action, future state, and value in parallel

### 6. Evaluation Metrics
- Success rate per task
- Overall success rate across all tasks
- Logs to file and optionally WandB
- Saves replay videos for visualization

## Configuration Parameters

### Key Parameters for LIBERO Evaluation

- `task_suite_name`: Which LIBERO suite (libero_10, libero_spatial, etc.)
- `num_trials_per_task`: Episodes per task (default: 50)
- `chunk_size`: Actions predicted per query (default: 16)
- `num_open_loop_steps`: Actions executed before requery (default: 16)
- `num_queries_best_of_n`: Number of queries for best-of-N (default: 1)
- `deterministic`: Whether to use fixed seeds for reproducibility
- `use_parallel_inference`: Enable multi-GPU parallel queries
- `num_denoising_steps_action`: Number of diffusion steps (default: 5)

## Notes

- The system is designed for reproducible evaluation with configurable parallelization
- Best-of-N search significantly improves success rates but requires more compute
- Action chunking enables efficient open-loop execution while maintaining flexibility
- The latent sequence structure allows the model to predict multiple outputs (actions, future state, value) in a single forward pass
